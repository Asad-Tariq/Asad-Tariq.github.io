## Introduction

This course explores in detail the theoretical and practical aspects of Relational Database Management Systems (RDBMS). It aims to build an understanding of database modeling, relational algebra, structured query language (SQL), transaction management, and concurrency control. Additionally, the course covers topics related to database connectivity with front-end applications and discusses administrative aspects of database systems. These aspects include database security, the distinction between database management, data warehousing, and data mining, as well as big data and its challenges. Modern database techniques are also a part of the course, including a discussion of MongoDB as one of the modern databases.

---

## Part 1: Foundations of Database Systems

This section introduces the core concepts and motivations behind database systems. Students learn about the relational data model, database schemas, and keys as the basis for structuring data. The section covers database design principles through the ER model, relationships, and integrity constraints, and then moves on to normalization up to 3NF to ensure efficient and consistent data organization.

**Learning Goal:** Students will understand the fundamental structures and design principles that underpin relational databases.

---

## Part 2: Query Languages and Data Manipulation

Here, students move from design to interaction with databases. Beginning with relational algebra, the section introduces SQL as the primary query language. Students learn to define and manipulate data using DDL and DML, including SELECT queries, aggregates, joins, groupings, subqueries, and data modification operations. This section ensures students can retrieve and update information effectively while appreciating the expressiveness and limitations of relational queries.

**Learning Goal:** Students will gain proficiency in formulating queries and manipulating data using SQL and relational operations.

---

## Part 3: Transactions, Concurrency, and Performance

This section explores the operational side of database systems. Students learn about transaction management, ACID properties, and the challenges of ensuring reliability in multi-user environments. Concurrency control topics such as serializability, locking protocols, deadlocks, and isolation levels are covered in detail. Finally, indexing is introduced as a key performance optimization technique.

**Learning Goal:** Students will understand how databases maintain consistency, reliability, and efficiency in multi-user environments.

---

## Part 4 (Conclusion): Advanced Topics and Applications

The final section looks beyond the relational model to modern database technologies. Students are introduced to NoSQL systems, database indexing in greater depth, and backup/restore strategies for data recovery. The course concludes with project demonstrations and review sessions, where students integrate and apply their knowledge to real-world database scenarios.

**Learning Goal:** Students will explore modern database techniques and apply their skills to practical projects.
