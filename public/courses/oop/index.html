Introduction#

Object Oriented Programming (OOP) offers a new and powerful way to cope with complexity. Instead of viewing a program as a series of steps to be carried out, it views it as a group of objects that have certain properties and can take certain action. This course aims to introduce fundamental OOP concepts, and use C++ as an object oriented programming language.

Part 1: Foundations of C++ and Programming Constructs#

This section establishes the groundwork by introducing students to the syntax and structure of C++. Core programming fundamentals such as data types, operators, conditionals, and loops are covered to ensure a solid base. Arrays, pointers, and dynamic memory allocation are introduced to give students insight into how data is stored and manipulated at a lower level. The section concludes with an exploration of functions, parameter passing techniques (by value, by reference), debugging practices, and the concept of function overloading. Together, these topics prepare students for the object-oriented principles that follow.

Part 2: Object-Oriented Programming Essentials#

With a foundation in C++ established, this section introduces the key principles of object-oriented programming. Students learn about classes and objects, as well as encapsulation through public and private members. The concepts of constructors, destructors, and the Rule of Five are introduced, highlighting proper resource management (RAII). This is followed by applications of data structures (Linked Lists, Stacks, Queues) and operator overloading. Finally, the section covers inheritance and polymorphism, allowing students to understand and implement code reuse, extension, and runtime flexibility in C++ applications.

Part 3: Advanced C++ Features and Design Principles#

This section moves beyond the basics of OOP into more advanced language features and design practices. Students learn about static members, friend functions and classes, and design patterns as reusable solutions to recurring software problems. Templates are introduced to enable generic programming, broadening the flexibility of C++ programs. By the end of this section, students will not only understand advanced features of the language but also begin to appreciate the design-level thinking required for building scalable and maintainable software.

Part 4 (Conclusion): Modeling, Application, and Integration#

The final section emphasizes applying object-oriented principles in larger contexts. Students study object relationships (association, composition, and aggregation) and represent them through class diagrams, bridging the gap between theoretical modeling and practical implementation. Activities reinforce translating real-world scenarios into structured designs. The section concludes with instructor-selected advanced topics in C++ and culminates in project demonstrations, where students showcase their ability to apply the concepts of the course to real applications.
